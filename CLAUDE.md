# CLAUDE.md — Правила для цього проекту

## Мова

- Вся документація ведеться **українською мовою**
- Код пишеться англійською, коментарі — українською

## Безпека

### Персональні дані користувачів
- Ніколи не логувати ім'я, номер телефону, населений пункт клієнта у відкритому вигляді
- Персональні дані зберігати лише в захищеній базі даних, не в коді та не в змінних середовища без шифрування
- Згода на обробку персональних даних є обов'язковою перед збором будь-якої інформації від клієнта

### Токени та секрети
- Telegram Bot Token та інші секрети зберігати виключно у `.env` файлі
- `.env` файл завжди має бути в `.gitignore` — ніколи не комітити його
- Не вставляти токени, паролі або API-ключі безпосередньо в код

### Валідація вхідних даних
- Перевіряти та очищати всі дані, що надходять від користувача через бота
- Номер телефону валідувати за форматом перед збереженням
- Не довіряти callback_data з Telegram без перевірки

### Доступ адміністратора
- Адмін-команди доступні лише для заздалегідь визначених Telegram ID
- ID адміністраторів зберігати в `.env`, не в коді
- Будь-яка адмін-дія має логуватися

### Залежності
- Не додавати нові пакети без явного дозволу
- Регулярно перевіряти вразливості: `npm audit` або `pip audit`

## Що не можна робити

- Не видаляти рядок з мовою інтерфейсу з документації
- Не комітити `.env`, `node_modules`, або файли з токенами
- Не додавати функціональність поза MVP без узгодження
