# Завдання для RESEARCHER — Пошук на GitHub

## Завдання #1 — Знайти актуальні приклади aiogram 3.x для оформлення замовлень

**Виконавець:** RESEARCHER  
**Статус:** TODO  
**Контекст:** Будуємо Telegram-бота для похоронного агентства. Основна функція MVP — оформлення замовлення: збір ім'я, телефону, населеного пункту, згода на обробку даних, підтвердження із номером заявки, сповіщення менеджеру. Технологія — Python + aiogram.

---

## Що шукати

### 1. Репозиторії з aiogram 3.x + FSM для збору даних форми / бронювання

Запити для пошуку на GitHub:
- `aiogram 3 FSM order booking 2025 OR 2026`
- `aiogram3 StatesGroup form booking telegram bot`
- `aiogram 3 inline keyboard order confirmation`

**Що потрібно знайти:**
- Як реалізований FSM (Finite State Machine) для покрокового збору даних від користувача (ім'я → телефон → місто → підтвердження)
- Як передається заявка в окремий чат або канал менеджера (`bot.send_message` до адмін-чату)
- Як генерується унікальний номер заявки
- Як клієнт може скасувати замовлення через inline-кнопку

### 2. Приклади обробки згоди на персональні дані в aiogram 3.x

Запити:
- `aiogram 3 privacy consent gdpr telegram bot`
- `aiogram 3 user agreement inline button`

**Що потрібно знайти:**
- Патерн підтвердження згоди через InlineKeyboardMarkup перед збором контактних даних

### 3. Критерії якості репозиторіїв

Брати до уваги лише ті, що:
- Мають коміти у 2025–2026 роках (активно підтримуються)
- Використовують `aiogram >= 3.0` (не aiogram 2.x)
- Мають `requirements.txt` або `pyproject.toml` з явною версією aiogram
- Бажано: є `.env.example`, є структура `handlers/`, `keyboards/`

### 4. Що виключати

- Репозиторії на aiogram 2.x
- Репозиторії без комітів після 2024 року
- Приклади без FSM (прості echo-боти)
